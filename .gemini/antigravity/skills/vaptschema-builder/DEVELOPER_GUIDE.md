# VAPTSchema Builder Developer Guide

## Architecture Context

The VAPTSchema-Builder skill exists at the intersection between the **AI Agent Layer** and the **PHP Driver Layer**. 

When the AI Agent outputs JSON based on this skill's rules, it gets saved to the database. At runtime, the `VAPT_Driver` parses this generated JSON alongside `vapt_driver_manifest_v2.0.json` to write physical configuration files (`.htaccess`, `wp-config.php`, etc.) to the server.

## Troubleshooting Driver Failures

If the JSON generated by this skill fails to apply on the server, the problem usually stems from how the driver interprets the manifest.

### 1. The Idempotency Check Skipped Execution
* **Symptom:** The driver reports "success" but the rule isn't in the file.
* **Cause:** The `idempotency.check_string` (like `# BEGIN VAPT RISK-003`) was already found in the file, and `if_found` was set to `skip`.
* **Fix:** Check if a previous failure left orphaned markers in the file.

### 2. Anchor String Not Found
* **Symptom:** Rule is inserted at the very end of the file instead of before `# BEGIN WordPress`.
* **Cause:** The target `.htaccess` file lacks the standard `# BEGIN WordPress` marker, forcing the driver to use the `insertion.fallback` strategy (usually `append`).
* **Fix:** Reset permalinks in WordPress to have it regenerate its canonical `.htaccess` block.

### 3. Mod_Rewrite 500 Errors
* **Symptom:** Applying the rule crashes the site with a 500 error.
* **Cause:** The generated JSON omitted the `<IfModule mod_rewrite.c>` wrapper or `RewriteEngine On` directives.
* **Fix:** Review the json against `enforcer_pattern_library_v2.0`.

## Adding a New Risk to the System

To teach this skill a new risk (e.g., `RISK-126`), you cannot just tell the AI to "generate it". The underlying 5-file bundle must be updated:
1. **Enforcer Pattern Library:** Add `RISK-126` code stubs here.
2. **Interface Schema:** Define the UI toggles, layout, and driver references here.
3. **Driver Manifest:** Add the raw execution steps (`write_mode`, `target_file`, etc.) here.

You can use the AI Agent to draft these objects, but they must be committed back to the VAPT-Secure `.data` bundle to actually execute.
