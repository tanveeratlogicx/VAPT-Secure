# Implementation Plan: Global Fix — Missing `<IfModule mod_headers.c>` Wrapper

**Scope expanded from user feedback**: Fix applies globally to all Header directives across all RISKs.

## Root Cause

`RISK-012` `.htaccess` rule was missing `<IfModule mod_headers.c>`. This is a global issue — any bare `Header` directive can cause `500 Internal Server Error` if `mod_headers` is not loaded.

The injection path is via `class-vaptsecure-apache-deployer.php` which reads `code`/`rules` fields verbatim.

## Changes Applied

### Layer 1 — Pattern Library (Source Data)
#### [MODIFY] enforcer_pattern_library_v2.0.json
- **33 bare Header entries** across **11 RISKs** globally updated
- Affected: RISK-012, RISK-014, RISK-015, RISK-022, RISK-024, RISK-031, RISK-032, RISK-033 (+ 3 more)
- Fields patched: `code`, `wrapped_code`, `write_block`

---

### Layer 2 — Live Server Config
#### [MODIFY] .htaccess
- RISK-012 rule immediately wrapped in `<IfModule mod_headers.c>`

---

### Layer 3 — Enforcer (Safety Net)
#### [MODIFY] class-vaptsecure-apache-deployer.php
- Added `ensure_ifmodule_header_wrapper()` defensive method
- Called from `extract_rules()` — auto-wraps any bare Header directives before injection
- Acts as a persistent safety net for all future deployments
